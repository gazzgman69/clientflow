3) Fix the provider catalog (DB-backed) + API

Create two tables and wire them end-to-end so the UI stops showing “Failed to load email provider configurations”.

Schema

email_providers

id (pk, uuid)

slug (text, unique)

display_name (text)

supports_incoming (bool)

supports_outgoing (bool)

incoming_auth (enum: 'oauth' | 'password' | 'none')

outgoing_auth (enum: 'oauth' | 'password' | 'none')

default_imap_host (text, nullable)

default_imap_port (int, nullable)

default_imap_secure (bool, nullable)

default_smtp_host (text, nullable)

default_smtp_port (int, nullable)

default_smtp_secure (bool, nullable)

help_text (text)

tenant_email_prefs

tenant_id (fk → tenants.id, pk)

bcc_self (bool, default false)

read_receipts (bool, default false)

show_on_dashboard (bool, default true)

updated_at (timestamp)

Backend endpoints

GET /api/email/providers → list email_providers

POST /api/email/providers/seed (dev-only) → seed the list below

GET /api/email/prefs (tenant-scoped) → returns tenant_email_prefs

PUT /api/email/prefs (tenant-scoped) → upsert prefs

Storage interface (must exist & be implemented)

getEmailProviders(): Promise<EmailProvider[]>

seedEmailProviders(providers: EmailProviderSeed[]): Promise<void>

getTenantEmailPrefs(tenantId: string): Promise<TenantEmailPrefs>

upsertTenantEmailPrefs(tenantId: string, prefs: Partial<TenantEmailPrefs>): Promise<void>
