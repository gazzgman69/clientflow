## Behavior
- Public flow (portal):
  - Steps: Selection → **Details (Extra Info)** → Review & Sign → Pay.
  - Show only the questions configured for that quote (standard + custom).
  - Validate types (email, phone, date dd/mm/yyyy, time HH:mm, select, checkbox, textarea, file, address).
  - **Autosave** as user types; Show progress and inline errors.
  - “Accept” is disabled until: one package selected, all required fields valid, typed-name + checkbox ticked.

- Admin flow:
  - In Quote Editor, add a **“Contract Details”** panel with:
    - Master toggle: “Require extra info for this quote”.
    - **Standard questions selector** (multi-select).
    - **Custom fields builder** (label, key, type, required, help_text, options).
    - Show mapping + token for each field.
    - Prefill fields from Contact/Event where possible.

- Contract rendering:
  - On acceptance, generate **contract_snapshot** by replacing tokens with collected answers and quote/contact/event values.

- Data syncing:
  - If a field is mapped to a CRM entity, update that entity on acceptance (and optionally earlier if configured).
  - Custom fields persist with the Quote.

- Degraded mode:
  - If “Require extra info” is OFF, skip the Details step.
