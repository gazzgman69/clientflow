Goal (CRM-first)

Create/Update events in CRM first → they appear in my CRM calendar UI immediately.

Then push those events to Google asynchronously.

Never delete CRM events because they’re missing in Google.

If Google is missing an event that CRM has, re-push (or unlink Google id), but do not delete the CRM event.

Assumptions (keep exactly as-is)

events table has: id, title, type, start_date, end_date, calendar_id, is_cancelled (boolean), provider_event_id (nullable).

“Google-owned” = provider_event_id IS NOT NULL.

“CRM-owned” = provider_event_id IS NULL or type = 'lead'.

Lead capture already creates a CRM event.

1) Sync Policy (tiny module)

Create server/services/syncPolicy.ts:

export const SyncPolicy = {
  isCrmOwned: (e: { provider_event_id: string | null; type: string }) =>
    !e.provider_event_id || e.type === 'lead',

  canDeleteLocallyFromProvider: (e: { provider_event_id: string | null; type: string }) =>
    !!e.provider_event_id && e.type !== 'lead', // Only Google-linked & not lead

  onProviderMissing: 'repush', // 'repush' or 'unlink'
};

2) Outbox to push to Google (no schema changes)

Create two files.

server/services/googleOutbox.ts

type Job = { eventId: string; attempt?: number };
const q: Job[] = [];

export const googleOutbox = {
  enqueue(job: Job) { q.push({ ...job, attempt: job.attempt ?? 0 }); },
  _take(): Job | undefined { return q.shift(); },
  _size(): number { return q.length; }
};


server/workers/googleOutboxWorker.ts

import { googleOutbox } from '@/server/services/googleOutbox';
import { storage } from '@/server/storage'; // your existing storage svc
import { googleApi } from '@/server/services/googleApi'; // wrapper you already use

function sleep(ms: number) { return new Promise(r => setTimeout(r, ms)); }

async function handleJob(job: { eventId: string; attempt: number }) {
  const ev = await storage.events.getById(job.eventId);
  if (!ev) return;

  try {
    if (!ev.provider_event_id) {
      const gId = await googleApi.createEvent(ev);
      await storage.events.update(ev.id, { provider_event_id: gId });
      console.info('INFO google.outbox.create', { eventId: ev.id, gEventId: gId });
    } else {
      await googleApi.updateEvent(ev.provider_event_id, ev);
      console.info('INFO google.outbox.update', { eventId: ev.id, gEventId: ev.provider_event_id });
    }
  } catch (err: any) {
    const next = job.attempt + 1;
    if (next <= 5) {
      console.warn('WARN google.outbox.retry', { eventId: ev.id, attempt: next, reason: err?.message });
      await sleep(2 ** next * 250);
      googleOutbox.enqueue({ eventId: ev.id, attempt: next });
    } else {
      console.warn('WARN google.outbox.gave_up', { eventId: ev.id, reason: err?.message });
    }
  }
}

export async function runGoogleOutboxWorker(loop = false) {
  do {
    const job = googleOutbox._take();
    if (!job) { await sleep(500); continue; }
    await handleJob(job);
  } while (loop);
}

4) Harden the Google sync job (no deletions of CRM events)

Open your Google sync job/service (search for “sync” + “provider_event_id”). In the reconcile deletions section, replace with:

import { SyncPolicy } from '@/server/services/syncPolicy';
import { googleOutbox } from '@/server/services/googleOutbox';

// crmEventsWithProviderId = list from DB where provider_event_id IS NOT NULL
for (const ev of crmEventsWithProviderId) {
  const existsInGoogle = googleIndex.has(ev.provider_event_id!); // built from provider feed

  if (!existsInGoogle) {
    console.info('INFO google.sync.provider_missing', { eventId: ev.id, gEventId: ev.provider_event_id });

    // NEVER delete CRM if it's CRM-owned or a lead
    if (SyncPolicy.isCrmOwned(ev)) {
      console.info('INFO google.sync.protect_crm', { eventId: ev.id, reason: 'crm-owned-or-lead' });
      // Optionally repush if it was previously linked but now missing:
      if (process.env.GOOGLE_SYNC_REPUSH_ON_MISSING === 'true') {
        googleOutbox.enqueue({ eventId: ev.id });
        console.info('INFO google.sync.repush_scheduled', { eventId: ev.id });
      }
      continue;
    }

    // Google-owned & not lead → unlink or repush (but DO NOT delete CRM row)
    if ((process.env.GOOGLE_SYNC_REPUSH_ON_MISSING ?? 'true') === 'true') {
      googleOutbox.enqueue({ eventId: ev.id });
      console.info('INFO google.sync.repush_scheduled', { eventId: ev.id });
    } else {
      await storage.events.update(ev.id, { provider_event_id: null });
      console.info('INFO google.sync.unlinked', { eventId: ev.id });
    }
  }
}

// Summary logging (keep your imports/updates as-is)

Acceptance Criteria

Creating a lead immediately shows a “Lead” event in the CRM calendar UI.

Outbox later sets provider_event_id (linked to Google).

Google sync does not delete CRM events if Google is missing them; it repushes or unlinks.

Logs show google.outbox.* and google.sync.* actions clearly.
