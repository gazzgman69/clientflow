Goal:
Do a comprehensive sweep and repair of naming conventions with full data backfill.

Database (Postgres): all columns must be snake_case

App code (TS/JS, API, React): all properties must be camelCase

ORM (Drizzle schema): must map snake_case DB fields → camelCase app fields

Data backfill: if both snake_case + camelCase exist in DB, migrate & backfill without data loss

Phase A — Discovery (Read-only)

Search the repo for schema definitions (shared/schema.ts, migrations, models).

List all DB columns that are camelCase (❌ e.g., tenantId, projectDate).

Show mismatches: file, line, column name, expected vs actual.

Phase B — Schema Normalisation

For each camelCase DB column:

Rename → snake_case:

Ensure NOT NULL / FK constraints preserved.

Backfill logic if duplicates exist:

If both columns exist (tenantId + tenant_id):

UPDATE projects SET tenant_id = tenantId WHERE tenant_id IS NULL AND tenantId IS NOT NULL;
ALTER TABLE projects DROP COLUMN tenantId;


Do this safely for each affected table.

Update Drizzle schemas to map correctly:

projectDate: date("project_date"),
tenantId: varchar("tenant_id"),
venuePhone: text("venue_phone"),
