Contacts-Only Ingestion:

If no matching contact found in the same tenant → skip storing (or store in a quarantine table).

Store contact_id with the email record.

Auto-Link Projects:

When a contact has one or more active projects in the same tenant, automatically set project_id on the email.

If multiple projects are found, link to the most recent active one (by created_at or status).

Example:

const projects = await storage.getActiveProjectsByContact(contact.id, tenantId);
const projectId = projects.length > 0 ? projects[0].id : null;

  tenantId,
  contactId: contact.id,
  projectId,
  ...parsedEmail
});


API Route Filtering:

Update /api/email/threads and /api/emails to **only return emails with contact_id IS NOT NULL`.

Backfill Migration:

Run SQL to attach contact_id to existing emails where possible:

UPDATE emails e
SET contact_id = c.id
FROM contacts c
WHERE e.tenant_id = c.tenant_id
  AND e.contact_id IS NULL
  AND e.from_email = c.email;


Then backfill project_id for any email linked to a contact with an active project:

UPDATE emails e
SET project_id = p.id
FROM projects p
WHERE e.contact_id = p.contact_id
  AND p.status = 'active'
  AND e.project_id IS NULL;


Quarantine or delete any remaining emails with contact_id IS NULL.
