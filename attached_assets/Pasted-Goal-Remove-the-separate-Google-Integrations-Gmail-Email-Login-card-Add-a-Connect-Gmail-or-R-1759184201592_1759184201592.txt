Goal

Remove the separate “Google Integrations → Gmail Email Login” card.

Add a Connect Gmail (or Reconnect/Disconnect) button row directly inside Settings → Email Settings → Incoming Email.

Keep the calendar section unchanged.

Preserve all existing OAuth + sync logic; we are only changing where the controls are rendered and how state is surfaced.

Acceptance criteria

The Incoming Email panel shows:

Status pill (Connected / Not connected), connected account email, last synced.

Primary action button: Connect Gmail (when not connected) or Reconnect Gmail (when connected).

Secondary Disconnect button when connected.

A mini “Last synced” timestamp and scope list on hover or subtext.

The old top-level Gmail Email Login card is completely removed from the page.

All actions call the existing endpoints:

Connect/Reconnect → GET /api/auth/google/gmail (OAuth start)

Disconnect → POST /api/auth/google/gmail/disconnect

Status → GET /api/auth/google/gmail/status

Multi-tenant safe: requests carry session and tenant middleware as they do today.

If provider is not Google (future: Office 365/SMTP), the Incoming Email block still renders, but shows a provider switch link (“Use Microsoft/IMAP instead”) without breaking Gmail flow.

No visual regressions to Outgoing Email, BCC toggle, Read Receipts toggle.

Unit test: render Incoming Email when connected vs not, buttons call correct handlers.
