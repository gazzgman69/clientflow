Goal
Replace placeholder provider connections with production OAuth connectors for:

Google Gmail (Gmail API for send/sync)

Microsoft 365 / Outlook (Microsoft Graph for send/sync)

Everything must be tenant-scoped, support send + contacts-only sync, and expose Connect / Reconnect / Disconnect / Status from Email Settings → Incoming Email. Outgoing email should use provider APIs (not raw SMTP) when available.

Create an index on (tenant_id, user_id, provider).

3) OAuth routes & services
3.1 Google Gmail

Scopes:

https://www.googleapis.com/auth/gmail.modify (read + labels + basic write)

https://www.googleapis.com/auth/gmail.send (send)

Routes:

GET /api/auth/google/gmail → start OAuth (state includes tenantId, userId, returnTo)

GET /api/auth/google/gmail/callback → exchange code, store tokens (per tenant/user), set status='connected'

GET /api/auth/google/gmail/status → {connected, accountEmail, lastSyncedAt, scopes}

POST /api/auth/google/gmail/disconnect → revoke, clear tokens, set status='disconnected'

Service:

gmailService.refreshIfNeeded(integration)

gmailService.sendEmail({to, subject, html, text, replyTo}) via gmail.users.messages.send

gmailService.syncContactsOnly({since, pageToken}):

List messages via gmail.users.messages.list

For each message, fetch headers; ingest only if sender/recipient matches an existing Contact in current tenant

Map threadId/messageId → store email rows with tenant_id, contact_id, project_id?

Save and reuse historyId/nextPageToken to do incremental sync

Token store encrypt/decrypt, and persist expires_at.

3.2 Microsoft 365 (Graph)

Scopes (delegated):

Mail.Read

Mail.Send

offline_access

(optional) MailboxSettings.Read for time zone / locale

Routes:

GET /api/auth/microsoft → start OAuth (PKCE recommended)

GET /api/auth/microsoft/callback → exchange token; store per tenant/user

GET /api/auth/microsoft/status

POST /api/auth/microsoft/disconnect

Service:

msService.refreshIfNeeded(integration)

msService.sendEmail() via POST /me/sendMail

msService.syncContactsOnly() via /me/messages?$select=internetMessageHeaders,from,toRecipients,ccRecipients,replyTo,receivedDateTime,conversationId,subject,bodyPreview

Important: both services must validate from-address on send; if mismatch, use provider default sender and set replyTo to requested address. Log a warning in the response the UI can surface.

5) Sending pipeline

Outgoing email route should dispatch by active provider:

If provider='google' → Gmail API

If provider='microsoft' → Graph API

Save a send log row per tenant/user with provider response (messageId/graph id), status, error (if any).

Respect BCC me + Read receipts toggles (store headers appropriately):

Gmail read receipts are limited; if unsupported, gracefully ignore and note in response.

For “BCC me”, add user’s email as BCC when sending (configurable per tenant).

6) Sync worker (contacts-only)

Create/extend a periodic job:

Enumerate email_provider_integrations where status='connected' per tenant.

For each, call the right service’s syncContactsOnly.

Contacts-only rule: only ingest emails where any participant matches a contact email in this tenant.

Link to the most recent open project for that contact when possible (fallback: contact only).

Do not ingest if no contact → skip; (optional) increment a “skipped_no_contact” metric.

Store nextSyncCursor (Gmail historyId / Graph deltaLink if you add delta later).

7) Security & tenancy

All routes behind ensureUserAuth, tenantResolver, requireTenant.

All storage calls must include tenantId; add asserts to throw if missing.

Encrypt tokens at rest; never return tokens to the client.

When disconnecting, attempt provider revoke and wipe local tokens.

11) Done criteria (visible)

In Settings → Email, I can choose Google or Microsoft and press Connect.

After OAuth, Incoming Email shows Connected, my account email, and Last synced.

I can send an email; provider returns message id; UI shows success.

The sync job brings in recent contact-linked emails only; dashboard email widgets show them.

Switching to the other provider works (prompts to disconnect the first).

All records store tenant_id and pass isolation checks.

Helpful implementation notes

Google sender: always the authenticated account; if UI requests a different From, set it as replyTo and return a warning.

Microsoft behaves similarly via Graph; use /me/sendMail.

Prefer incremental cursors (Gmail historyId, Graph delta) to minimize API cost; add exponential backoff on 429/5xx.
