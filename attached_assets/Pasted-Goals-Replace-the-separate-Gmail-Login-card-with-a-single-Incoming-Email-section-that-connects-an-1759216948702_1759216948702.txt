2) Server routes (auth + tenant middleware on all)
Catalog

GET /api/email/providers → list from email_provider_catalog (no secrets).

Connect (Incoming Email)

OAuth providers

GET /api/email/connect/:providerKey → redirect to provider auth (google/microsoft).

GET /api/email/callback/:providerKey → exchange code, store encrypted tokens in email_accounts, set status='connected', account_email, expires_at.

IMAP/SMTP providers

POST /api/email/connect/:providerKey with body:

{ username, password, custom?: {imapHost,imapPort,imapSecure,smtpHost,smtpPort,smtpSecure} }


Validate by:

Try SMTP EHLO + AUTH (nodemailer transport) then NOOP, and IMAP LOGIN then LIST root.

On success, save encrypted username/password, status='connected'. On fail, return {code, providerMessage}.

Status / Disconnect

GET /api/email/status → {connectedProviders:[...], active:{providerKey,accountEmail,lastSyncedAt}}

POST /api/email/disconnect/:providerKey → revoke OAuth (where applicable), delete secrets, set status='disconnected'.

Send / Sync

POST /api/email/send → dispatcher (see §4) sends via active account; respects tenant prefs (BCC me, read receipts flag header).

POST /api/email/sync → triggers contacts-only ingestion for the active account.

Debug

POST /api/email/debug/send-test → {to,subject,body,providerKey?}; if missing, use active. Return provider id/accepted + provider diagnostic.

GET /api/email/debug/health/:providerKey → attempts small IMAP/SMTP login or token refresh and reports.

3) Provider services

Create services/emailProviders/:

google.ts (OAuth Gmail): send via gmail.users.messages.send, incremental sync using historyId; contacts-only filter.

microsoft.ts (Graph): POST /me/sendMail, paging on /me/messages; contacts-only filter.

imapSmtp.ts: generic:

send using nodemailer SMTP (TLS/STARTTLS based on recipe).

sync via IMAP (node-imap/imapflow): select INBOX, fetch headers/bodies; only ingest messages where any participant matches a Contact email in this tenant; store cursor (UIDVALIDITY + last UID).

Support IDLE or periodic poll (configurable).

All services must:

encrypt/decrypt secrets via ENCRYPTION_KEY

refresh tokens where applicable

never return secrets to the client

return structured results:

{ ok: true, provider: 'google'|'microsoft'|'imap_smtp', messageId?, internetMessageId?, warnings?[] }
