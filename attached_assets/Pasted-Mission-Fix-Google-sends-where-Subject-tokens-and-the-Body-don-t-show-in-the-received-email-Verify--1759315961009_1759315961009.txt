Verify the exact endpoint the UI calls, confirm payload fields, and ensure the server’s send handler normalizes → interpolates tokens ([FirstName], [X|default]) → passes subject, html, and text into the Gmail sender with a correct MIME. Keep OAuth/DB unchanged. Microsoft path untouched.

Rules

Keep edits minimal and scoped to the send route + helpers.

Never log raw email content; only lengths and a short preview when explicitly asked.

If alternative SMTP/Nodemailer path is used, wire subject/html/text into sendMail.

3) Add/ensure a normalization + token layer (only if missing)

If not present, add:

server/src/lib/payload-normalize.ts with normalizeSendPayload(body) that maps many client field names → { to[], cc[], bcc[], subject, html?, text?, attachments?, tokens? }.

server/src/lib/merge.ts with:

interpolateTokens(input, ctx) supporting square brackets [FirstName], [FirstName|there] (case-insensitive, dotted paths allowed), and legacy {{...}}.

buildSendContext({ to, accountEmail, user, company, contact, tokens }).

stripHtml(html) helper.

In the handler, build:

const norm = normalizeSendPayload(req.body);
const ctx  = buildSendContext({ to: norm.to, accountEmail: account.account_email, /* add user/company/contact if available */, tokens: norm.tokens });

const finalSubject = interpolateTokens(norm.subject, ctx);
const htmlBody     = interpolateTokens(norm.html, ctx);
const textBody     = interpolateTokens(norm.text, ctx) || (htmlBody ? stripHtml(htmlBody) : '');


Add a second short log:

4) Verify which sender is used and wire subject/html/text

If using Gmail REST (googleapis.gmail.users.messages.send), ensure you call a MIME builder that inserts:

headers (MIME-Version, Subject encoded UTF-8, Date, To…),

blank line,

multipart/alternative with text/plain (quoted-printable) and text/html (quoted-printable),

optional attachments as multipart/mixed.

Then base64url encode and send as requestBody.raw.

If using Nodemailer SMTP: ensure sendMail({ subject: finalSubject, html: htmlBody || undefined, text: textBody || undefined, ... }).

If such a builder isn’t used, create server/src/services/gmail-send.ts (as needed) and use it. Otherwise, prove the existing builder is passed htmlBody/textBody and not empty strings.

5) Add a dry-run preview endpoint (behind DEBUG_EMAIL=1)

In the same routes file, add:

if (process.env.DEBUG_EMAIL === '1') {
  app.post('/api/dev/email/preview', (req, res) => {
    const norm = normalizeSendPayload(req.body);
    const ctx  = buildSendContext({ to: norm.to, contact: { email: norm.to?.[0] }, tokens: norm.tokens });
    const subj = interpolateTokens(norm.subject, ctx);
    const html = interpolateTokens(norm.html, ctx);
    const text = interpolateTokens(norm.text, ctx) || (html ? stripHtml(html) : '');

    // Use the same MIME function the real send uses:
    const { buildMimeRaw } = require('../services/gmail-send');
    const raw = buildMimeRaw({
      from: 'preview@example.com',
      to: norm.to?.length ? norm.to : ['them@example.com'],
      subject: subj || 'Subject',
      html: html || '<p>Empty</p>',
      text: text || 'Empty'
    });

    const decoded = Buffer.from(raw.replace(/-/g, '+').replace(/_/g, '/'), 'base64').toString('utf8');
    return res.json({
      finalSubject: subj,
      htmlLen: (html || '').length,
      textLen: (text || '').length,
      preview: decoded.slice(0, 800) // headers + start of multipart body
    });
  });
}
