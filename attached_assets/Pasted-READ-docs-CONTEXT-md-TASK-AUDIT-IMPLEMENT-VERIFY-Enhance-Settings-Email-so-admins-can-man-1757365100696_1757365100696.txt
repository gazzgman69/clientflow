DB (extend mail_settings)
  - fromName text, fromEmail text, replyToEmail text
  - isActive boolean default true
  - isDefault boolean default true (only one allowed)
  - syncIntervalMinutes int default 5 (clamp 1–15)
  - lastTestedAt timestamptz nullable
  - lastTestResult text nullable
  - lastTestError text nullable
  - quotaUsed int default 0
  - quotaLimit int default 500 (daily default; configurable later)

Logs table (mail_settings_audit)
  - id uuid pk
  - settingsId fk → mail_settings.id
  - kind text (‘imapTest’, ‘smtpTest’, ‘sync’, ‘send’, ‘quota’)
  - ok boolean
  - error text nullable
  - durationMs int
  - createdAt timestamptz default now()
  - meta jsonb
  INDEX(settingsId, createdAt DESC)

Backend routes (/api/settings/mail)
- GET  /current → settings with redacted passwords
- POST /save → save encrypted creds, enforce 1 default, auto-test → update lastTested*
- POST /test → run IMAP+SMTP connect (5s timeout), update lastTested*, log results
- POST /detect → MX lookup by email → return suggested hosts/ports
- GET  /logs?limit=20 → recent audit rows
- POST /send-test → send a short test email to fromEmail (logs as kind=‘send’)

*lastTested fields: update lastTestedAt, lastTestResult (‘ok’/’fail’), lastTestError

Quota Monitoring
- smtp.ts: increment quotaUsed on send
- If quotaUsed > 80% of quotaLimit → log warning + surface UI banner
- If quotaUsed exceeds quotaLimit → block send, clear error

Alerts
- If sync fails 3+ times consecutively:
  - Store counter in audit/logs
  - Trigger notification: email (to admin), SMS (if phone configured)
  - Reset counter on success
- Alerts never include passwords

Frontend (EmailSettings.tsx / MailForm.tsx)
- Provider Presets dropdown + Auto-Detect button
- Fields: fromName, fromEmail, replyTo
- Toggles: Enable/Disable, Make Default
- Sync interval selector (1–15 mins)
- Buttons: Test, Save, Send Test Email
- Status line: “Last tested: dd/MM/yyyy HH:mm — OK/Failed”
- Logs section: show last 20 attempts
- Soft alerts: inline banner if repeated failures detected

Frontend (Dashboard widget)
- Show Last Checked, Last Result, Next Sync
- Badge for quota (e.g., 420/500 used; turns orange/red on thresholds)
- Banner if sync disabled or failing
