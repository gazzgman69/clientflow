GOAL
Move all email credential management into Settings and remove the standalone Email tab:
1) DELETE the “Email” tab/item from the left-hand navigation.
2) ADD a “Settings → Email Login” page with:
   - Connection status for Google (connected/disconnected + scopes)
   - Button: “Connect / Reconnect Google” → /auth/google
   - Button: “Disconnect Google” (calls POST /api/auth/google/disconnect)
   - (Optional, scaffold only) Hidden/disabled SMTP form placeholder for future.

SCOPE
You may create/edit ONLY these files (create if missing):
  - server/src/routes/oauth.ts             // add status + disconnect if missing
  - server/src/services/google-oauth.ts    // reuse existing helpers; don't widen scopes here
  - web/src/components/Sidebar.tsx         // or whatever file renders the left nav; remove Email item
  - web/src/pages/settings/EmailLogin.tsx  // new page
  - web/src/pages/settings/Settings.tsx    // or settings index/nav; add link to Email Login
  - web/src/router.tsx or web/src/App.tsx  // wherever routes are registered; remove /email route if present, add /settings/email-login

RULES
- Do NOT add new dependencies or change DB schema.
- Do NOT remove any backend email send/list code; this task is ONLY navigation + settings UI + small auth endpoints.
- If file names differ (e.g., Navigation.tsx instead of Sidebar.tsx), locate the correct file and list it in the plan before changing it.

BACKEND REQUIREMENTS
A) Status endpoint:
   GET /api/auth/google/status  -> { ok:true, connected:boolean, scopes:string[] }
   - Determine connected if stored token exists for req.user.id.
   - If scopes are not stored, return an empty array or derive from the token’s scope string when available.

B) Disconnect endpoint:
   POST /api/auth/google/disconnect -> { ok:true }
   - Clears the stored Google tokens for req.user.id using existing storage.
   - Must NOT affect other providers or user data.

FRONTEND REQUIREMENTS
C) Remove Email tab:
   - Find the left-hand navigation component and DELETE the “Email” menu item/link.
   - Remove/disable the /email page route so no dead link remains.

D) Settings → Email Login page:
   - New page at /settings/email-login using shadcn/ui.
   - On mount, fetch /api/auth/google/status.
   - Show a Card with:
     • Status badge: Connected / Disconnected
     • If disconnected or missing Gmail scopes (gmail.send & gmail.readonly), show an Alert with:
       “Google email access required — please reconnect to grant Gmail permissions.”
     • Button “Connect / Reconnect Google” → navigates to /auth/google (opens OAuth)
     • Button “Disconnect Google” → POST /api/auth/google/disconnect then refetch status
   - OPTIONAL: Add a collapsed/disabled SMTP section (commented “coming soon”), no inputs wired yet.

E) Settings navigation:
   - In the Settings landing or sidebar, add a link item: “Email Login” → /settings/email-login.

UX DETAILS
- Use Tailwind + shadcn components (Card, Button, Badge/Alert).
- Keep styling consistent with existing Settings pages.
- Dates not required here; no date formatting changes.

   - “Connect / Reconnect Google” goes to Google consent.
   - “Disconnect Google” clears tokens and flips status to Disconnected.
