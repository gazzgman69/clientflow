SERVICE — emailSync.ts (NEW)
Export:
- parseHeaders(headers) → { messageId, inReplyTo, references[] }
- resolveThread({ subject, inReplyTo, references[], providerThreadId, projectId? }):
    1) if inReplyTo/references match existing emails.messageId → use that thread
    2) else if providerThreadId maps to any email → use that thread
    3) else create a new thread (subject || "(no subject)")
- upsertMessage(input):
    input = {provider, providerMessageId, providerThreadId, messageId, inReplyTo, references,
             direction, from, to[], cc[], bcc[], subject, snippet, sentAt, bodyHtml?, bodyText?,
             hasAttachments, projectId?, contactId?}
    Steps:
      a) threadId = resolveThread(...)
      b) insert emails row (ignore on UNIQUE clash)
      c) update email_threads.lastMessageAt = sentAt
      d) if projectId provided, set on email + thread if null
    Return { emailId, threadId }
- markThreadRead({ threadId, userId, when = new Date() })
- listThreads(projectId, { limit, offset }) → [{ id, subject, lastMessageAt, unreadCount, lastSnippet }]
- listMessages(threadId) → messages sorted ASC
Notes:
- unreadCount = messages after user.lastReadAt (if no record → count all).

SERVICE — gmail.ts (UPDATE)
- On SEND (existing send path):
    • Build RFC822, send via Gmail API.
    • After success, call emailSync.upsertMessage({ direction:'outbound', ... }).
    • If attachments present, persist with email_attachments (see attachments service).
- On FETCH (sync job or button):
    • List recent Gmail msgs for the user; for each not in DB by providerMessageId:
        – fetch headers + payload (HTML/text + attachment metadata)
        – call emailSync.upsertMessage({ direction:'inbound', ... })
        – if attachments exist, download/save metadata (actual file download optional now; set hasAttachments=true).
- Keep tokens via existing storage method; reuse your Google OAuth refresh.

ROUTES — emailUpload.ts (NEW)
- POST /api/email/upload
   multipart (multer), fields: files[]
   → returns [{ tempId, filename, size, mimeType, storageKey }]
   (Temp attachments will be associated when message is sent)
- GET /api/email/attachments/:id/download
   → streams file by storageKey, sets Content-Type, Content-Disposition

ROUTES — emailThreads.ts (NEW)
- GET /api/email/threads?projectId=...&limit=50
  → emailSync.listThreads()
- GET /api/email/threads/:id
  → { thread, messages: emailSync.listMessages(id) }
- POST /api/email/threads/:id/read
  Body: { when? } → emailSync.markThreadRead({threadId, userId, when})
- POST /api/email/compose
  Body: { projectId, threadId?, to[], cc?, bcc?, subject?, text?, html?, attachments?: [{storageKey, filename, mimeType}] }
  Logic:
    • If threadId present: set In-Reply-To to last message’s messageId and include References chain.
    • Build and SEND via Gmail (include attachments).
    • Persist outbound via emailSync.upsertMessage (set hasAttachments; insert email_attachments rows).
    • Return { ok:true, emailId, threadId }
